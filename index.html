About

소개

가설-지표-평가의 개선 플라이휠을 돌리며 한계 성능을 쥐어짜는 과정에 희열을 느낍니다. Advanced RAG 파이프라인(Bidcoin)에서는 LLM-as-Judge 평가를 11회 반복하며 답변 정확도를 0.52에서 0.78로 끌어올렸고, 생성형 AI 서비스(Go_Gachi)에서는 프로덕션 실측 로그를 근거로 응답 페이로드를 99.97% 줄였습니다.

이 집착은 4년간 육군 특수임무대 팀장으로 돌발 상황을 통제하고 대응 매뉴얼을 고쳐 쓰며 몸에 익힌 리스크 관리에서 비롯됐습니다. 지금은 AI 품질 모니터링 서비스 AIM을 혼자 만들어 실서비스로 운영하며 수명주기 전체를 완주하고 있습니다. 이 페이지의 모든 숫자는 실측입니다.

2026.06 — 현재 · 1인 개발

“이번 배포, 이전보다 정말 나아졌을까?” — 배포 후 웹서비스의 품질 변화를 근거 기반으로 판단하는 AI 품질 모니터링 플랫폼

  • 점수·등급과 장애 판정은 결정론적 코드로 계산하고, LLM은 수집된 근거를 서술하는 역할로 제한해 판정 결과의 재현성을 확보했습니다. LLM 호출 실패 시 규칙 기반 리포트로 자동 전환하는 폴백 경로도 갖췄습니다.
  • 가용성 · SSL · Lighthouse · Playwright 시나리오를 한 번에 검사하는 Celery · Redis 비동기 파이프라인을 구축하고, 작업 완료 순서와 무관하게 점수 · 회귀 비교 · 리포트가 일관되게 수렴하도록 정합성을 설계했습니다.
  • 시작 3주 차에 자체 도메인 HTTPS 실서비스에 도달했습니다 — 커밋 269 · 머지 PR 115 · 테스트 460개(백엔드 337 + 웹 123, 2026.07 실측).
  • 일일 오프박스 백업과 복원 리허설로 검증한 복구 런북을 갖추고, 운영 중 GCP → Oracle Cloud 이전을 수행하며 백업·복원 경로를 실전 재검증했습니다.
  • CI 배포 훅과 Slack/Discord webhook 알림을 연동하고, AIM 자신을 첫 프로젝트로 등록해 60분 주기로 스스로를 모니터링합니다.
